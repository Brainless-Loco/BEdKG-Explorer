5. How to read these patterns

# Level members:
#   - use qb4o:LevelMember
#   - point to their level with qb4o:memberOf
#   - store codes/names/roll-up links with mdAttribute:*
#
# Institute members:
#   - are also qb4o:LevelMember instances
#   - carry roll-up links such as inMouza, inThana, inInstituteType
#   - may also carry descriptive attributes like name, EIIN, email
#
# Observations:
#   - use qb:Observation
#   - point to a dataset with qb:dataSet
#   - attach dimensional coordinates with mdProperty:* predicates
#   - attach the fact value using a measure predicate such as isParticipating


6. File mapping in your project

# ABox.ttl
#   Core members such as institutes, mouzas, thanas
#
# ABox_MissingDimMembers.ttl
#   Formal qb4o:LevelMember declarations for dimensions like managementType,
#   educationLevel, instituteType
#
# ABox_InstituteAttrs.ttl
#   Extra descriptive/static attributes for institute members
#
# ABox_*_Obs.ttl
#   Fact/observation rows for each cuboid
